\section{Experimental Evaluation}
\label{sec:experiment}

To rigorously evaluate the efficacy of VulMorph-Fed, we design experiments to answer the following Research Questions (RQs):
\begin{itemize}
    \item \textbf{RQ1 (Cross-Project Generalisation):} How does VulMorph-Fed compare to state-of-the-art centralised and federated vulnerability detection baselines across large-scale, diverse, and rigorous cross-project evaluation benchmarks?
    \item \textbf{RQ2 (Ablation Study):} What are the individual contributions of VCSA, MCFPA, and MGMP to the framework's overall detection performance?
    \item \textbf{RQ3 (Privacy-Utility Trade-off):} How resilient is VulMorph-Fed to varying levels of Differential Privacy budgets ($\varepsilon$), and how does morphological abstraction benefit this trade-off?
    \item \textbf{RQ4 (Scalability):} How does the framework perform and scale as the number of federated clients increases?
\end{itemize}

\subsection{Experimental Setup}

\subsubsection{Datasets and Preprocessing}
To ensure a robust evaluation of cross-project generalisation—the primary failure mode of current models \cite{daoudi2024realvul}—we evaluate VulMorph-Fed across eight distinct vulnerability datasets representing the state of the art in SVD benchmarking:
\begin{enumerate}
    \item \textbf{Devign} \cite{zhou2019devign}: The canonical C/C++ dataset derived from FFmpeg, QEMU, Wireshark, and Linux kernel.
    \item \textbf{BigVul}: A massive C/C++ dataset mapping CVEs to GitHub commits.
    \item \textbf{ReVeal} \cite{chakraborty2022reveal}: Chromium and Debian codebase functions exhibiting realistic vulnerability class imbalance ($\approx 9.85\%$).
    \item \textbf{CVEfixes}: A multi-language dataset of vulnerabilities and fixes extracted directly from open-source CVE reports.
    \item \textbf{DiverseVul} \cite{chen2023diversevul}: Contains functions spanning 150 CWE types from over 295 GitHub projects.
    \item \textbf{MegaVul}: An extended variant of BigVul encompassing a broader historical timeframe.
    \item \textbf{PrimeVul} \cite{ding2025primevul}: The most rigorous recent benchmark featuring paired vulnerable/patched functions and a strict temporal split to prevent data leakage.
    \item \textbf{VulGate}: A synthetic/hybrid dataset designed to test adversarial robustness and extreme cross-domain transfer.
\end{enumerate}

For all datasets, we enforce a strict \emph{Cross-Project Federated Split}. Projects are partitioned such that the test set consists entirely of repositories unseen by any federated client during the training phase.

\subsubsection{Baselines}
We compare VulMorph-Fed against representative SOTA baselines:
\begin{itemize}
    \item \textbf{Centralised GGNN} \cite{zhou2019devign}: The standard graph baseline trained on globally pooled data without privacy constraints.
    \item \textbf{Centralised GAT} \cite{zhang2023cpvd}: A Graph Attention Network representative of modern domain-adaptation approaches.
    \item \textbf{FedAvg + GAT} \cite{yamamoto2023flr, zhang2024flvul}: A standard federated learning baseline employing the canonical FedAvg parameter aggregation.
\end{itemize}

\subsubsection{Implementation Details}
All experiments are implemented using PyTorch and PyTorch Geometric. VulMorph-Fed employs a 2-layer GNN with a hidden dimension of 128 and an embedding dimension of 64. The global CWE vocabulary spans $C=10$ classes (to simulate diverse vulnerability distributions across clients). Federated training runs for 10 global communication rounds, with each client performing 2 local epochs per round using the Adam optimizer with a learning rate of $10^{-3}$ and a batch size of 64. The default privacy budget is set to $\varepsilon = 2.0$. For evaluation metrics, we report F1-Score, Area Under the ROC Curve (AUC), Precision, and Recall.

\subsection{Results and Analysis}

\subsubsection{RQ1: Cross-Project Generalisation}
Table \ref{tab:rq1_main} and Figure \ref{fig:rq1_performance} present the cross-project F1-Scores of VulMorph-Fed compared to the baselines across all eight datasets.

The results strongly corroborate recent findings by \cite{weissberg2026codemetrics} and \cite{widyasari2026vultrial} that models relying on raw lexical tokens fail to generalize to unseen projects. Centralised GGNN and Centralised GAT consistently struggle, achieving an average F1-Score of only $\approx 0.35$ to $0.45$. Performance is particularly abysmal on PrimeVul (F1 $\approx 0.15$), highlighting the extreme difficulty of this temporally-split benchmark. Furthermore, standard FedAvg + GAT exacerbates the issue by suffering from client drift under non-IID graph splits.

In contrast, \textbf{VulMorph-Fed significantly outperforms all baselines across every single dataset}. By abstracting away lexical tokens via VCSA and federating CWE-conditioned structural prototypes, VulMorph-Fed successfully isolates invariant vulnerability morphologies, effectively closing the cross-project generalisation gap.

\subsubsection{RQ2: Ablation Study}
To isolate the contributions of the proposed components, we evaluate several variants of VulMorph-Fed on a representative dataset (DiverseVul). As shown in Table \ref{tab:rq2_ablation}, removing the Vulnerability-Critical Subgraph Abstraction (VCSA) or the Morphological Abstraction mapping results in severe performance degradation (F1 drops from 0.582 to 0.412 and 0.395, respectively). This mathematically confirms that feeding full, lexically dense CPGs into the federated pipeline overwhelms the GNN encoders with project-specific noise.

Similarly, replacing MCFPA with a uniform average or substituting MGMP with standard GAT message passing degrades the F1 score. MCFPA is crucial for weighting client contributions by semantic relevance, while MGMP actively injects the global structural consensus back into the local feature representations. 

\subsubsection{RQ3: Privacy-Utility Trade-off}
Table \ref{tab:rq3_privacy} and Figure \ref{fig:rq3_privacy} illustrate the impact of varying the Differential Privacy budget $\varepsilon$. Standard DP mechanisms applied to raw GNN parameters often destroy model utility at lower $\varepsilon$ values due to the massive dimension of the gradient vectors. However, VulMorph-Fed maintains highly competitive performance even under strict privacy constraints (e.g., F1=0.515 at $\varepsilon=0.5$). By reducing the local representation from thousands of unique vocabulary tokens down to just 8 abstract semantic types prior to prototype computation, VulMorph-Fed drastically limits the global sensitivity $\Delta f$, requiring exponentially less Laplace noise to satisfy DP guarantees.

\subsubsection{RQ4: Scalability}
Table \ref{tab:rq4_scalability} and Figure \ref{fig:rq4_scalability} evaluate VulMorph-Fed's performance and communication overhead as the number of federated clients $K$ increases from 3 to 50. The F1 score actually improves slightly and stabilizes near 0.595 as the global prototype bank observes a wider variety of code morphologies. Crucially, the Communication Cost per Round (CCR) remains under 300 Kilobytes (KB). Because VulMorph-Fed transmits compact prototype vectors $(C \times d)$ rather than full neural network weights ($O(|\theta|)$), the communication footprint is orders of magnitude smaller than standard FedAvg implementations (which transmit hundreds of Megabytes per round). This extreme efficiency makes VulMorph-Fed highly practical for global scale deployment.
